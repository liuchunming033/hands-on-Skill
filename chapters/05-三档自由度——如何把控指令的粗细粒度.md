
> 学习目标：解决指令"写太细 AI 变笨、写太粗 AI 跑偏"的两难问题

---

## 引言

写 Skill 最难把握的就是自由度——

写太细，AI 像个机器人不会变通；写太粗，AI 又容易跑偏。

本节教你根据任务风险来匹配指令的粗细粒度。

---

## 📊 三档自由度总览

根据任务的脆弱性和可变性，匹配指令的具体程度：

| 自由度档位 | 具体程度 | 适用场景 | 典型示例 |
|----------|---------|---------|---------|
| **高自由度** | 文本指令 | 多种方法都有效 | 代码审查、报告分析 |
| **中等自由度** | 伪代码/参数 | 存在首选模式 | 报告生成、数据处理 |
| **低自由度** | 固定脚本 | 必须遵循特定顺序 | 数据库迁移、部署流程 |

---

## 🔍 逐档拆解

### 高自由度：文本指令

**适用场景**：
- 多种方法都有效
- 决策取决于上下文
- 启发式方法指导处理方式

**典型任务**：
- 代码审查：检查结构、发现潜在问题、建议改进
- 报告分析：识别关键信息、提取要点、综合观点
- 问题诊断：分析症状、查找原因、提出方案

**写法示例**：
```markdown
## Code review process

1. Analyze the code structure and organization
2. Check for potential bugs or edge cases
3. Suggest improvements for readability and maintainability
4. Verify adherence to project conventions
```

**特点**：
- 只给出目标，不约束路径
- AI 可以根据上下文灵活选择方法
- 适合需要判断力的任务

---

### 中等自由度：伪代码/参数

**适用场景**：
- 存在首选模式
- 允许一定程度的变化
- 配置会影响行为

**典型任务**：
- 报告生成：有固定结构，但允许填充不同内容
- 数据处理：有标准流程，但参数可调
- 模板填充：模板固定，但变量可替换

**写法示例**：
````markdown
## Generate report

Use this template and customize as needed:

```python
def generate_report(data, format="markdown", include_charts=True):
    # Process data
    # Generate output in specified format
    # Optionally include visualizations
```
````

**特点**：
- 给出结构和参数
- AI 可以调整参数，但不能改变结构
- 适合需要标准化输出的任务

---

### 低自由度：固定脚本

**适用场景**：
- 操作脆弱且容易出错
- 一致性至关重要
- 必须遵循特定顺序

**典型任务**：
- 数据库迁移：必须按特定顺序执行，不能出错
- 部署流程：必须遵循检查清单，不能跳步骤
- 危险操作：删资源、改配置、切流量，一旦失控代价很高

**写法示例**：
````markdown
## Database migration

Run exactly this script:

```bash
python scripts/migrate.py --verify --backup
```

Do not modify the command or add additional flags.
````

**特点**：
- 给出精确指令，不允许变通
- AI 只需要执行，不能修改
- 适合高风险、必须一致性的任务

---

## 🎯 桥梁类比：理解自由度选择

想象 Agent 是一个探索路径的机器人：

### 两侧都是悬崖的狭窄桥梁（低自由度）

**场景**：只有一条安全的前进道路。

**应对**：提供具体的防护措施和精确的指令。

**例子**：数据库迁移，必须按精确顺序运行，一步错就崩溃。

---

### 没有危险的开阔地带（高自由度）

**场景**：许多路径都能通向成功。

**应对**：给出大致方向，相信 Agent 能找到最佳路线。

**例子**：代码审查，上下文决定最佳方法，多条路径都能成功。

---

## 📊 决策树：如何选择自由度？

### 1. 这个任务有唯一正确方法吗？
- **Yes** → **低自由度**（固定脚本）
- **No** → 继续判断

### 2. 有多种方法但存在最佳模式吗？
- **Yes** → **中等自由度**（伪代码/参数）
- **No** → **高自由度**（文本指令）

## 🚨 常见错误

### 错误 1：高风险任务给高自由度

**场景**：数据库迁移任务

**错误写法**：
```markdown
## Database migration

Please migrate the database carefully, verify the steps,
and make sure to backup before proceeding.
```

**问题**：
- AI 可能跳步骤
- AI 可能添加额外的操作
- 一旦出错，代价极高

**正确写法**：
````markdown
## Database migration

Run exactly this script:

```bash
python scripts/migrate.py --verify --backup
```

Do not modify the command or add additional flags.
````

---

### 错误 2：低风险任务给低自由度

**场景**：代码审查任务

**错误写法**：
```markdown
## Code review

Step 1: Check if the code follows naming conventions
Step 2: Check if there are unused imports
Step 3: Check if there are potential bugs
Step 4: ...
```

**问题**：
- AI 变成机械检查器，失去判断力
- 无法根据上下文灵活调整审查重点
- 效率低下，质量不佳

**正确写法**：
```markdown
## Code review process

1. Analyze the code structure and organization
2. Check for potential bugs or edge cases
3. Suggest improvements for readability and maintainability
4. Verify adherence to project conventions
```

---

## 🎯 本节核心观点

**自由度匹配原则：**
- 高风险、必须一致性 → **低自由度**（固定脚本）
- 存在最佳模式、允许参数调整 → **中等自由度**（伪代码/参数）
- 多种方法有效、需要判断力 → **高自由度**（文本指令）

**错误匹配 = 效率降低 + 风险增加**
