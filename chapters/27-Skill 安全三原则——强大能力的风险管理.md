
> 学习目标：理解Skill 安全三原则的核心概念和实践方法

---


## 引言

Skill 可以运行脚本、访问文件系统、甚至连接外部网络。

强大能力也意味着风险。

但你不能因为风险就不敢用 Skill，那样就失去了能力提升的机会。

**安全哲学**：安全不是限制能力，而是保护信任。

Anthropic 提出了三条安全建议，帮你在享受 Skill 强大能力的同时，管理好风险。

## 核心内容

### Skill 的强大能力

Skill 不是普通的文档，它可以让 Agent：

- **运行脚本**：可以执行任意代码
- **访问文件系统**：可以读取、修改、删除文件
- **连接外部网络**：可以访问外部 API、数据库、网络资源

这些能力是双刃剑。

### 安全风险来源

#### 风险类型

- **恶意代码**：脚本包含恶意逻辑
- **数据泄露**：访问敏感文件并上传
- **权限滥用**：执行超出预期的操作
- **供应链攻击**：依赖的外部资源被篡改

### 三条安全建议

#### 第一条：只安装可信来源的技能

**来源审查**：

- 官方提供的 Skill
- 知名开源社区的 Skill
- 企业内部开发的 Skill
- 经过安全审计的 Skill

**避免来源**：

- 不明来源的第三方 Skill
- 没有代码审查的 Skill
- 缺少维护和更新的 Skill

**类比**：就像手机应用，只从官方应用商店下载，查看权限申请。

#### 第二条：使用前仔细阅读所有文件

**重点审查**：

- **脚本依赖**：需要哪些外部库和工具
- **外部资源链接**：访问哪些外部服务和 API
- **文件访问路径**：读取和修改哪些文件
- **网络请求**：发送什么数据到哪个地址

**审查方法**：

- 阅读所有 `.py`、`.sh` 脚本文件
- 检查所有 `.json` 配置文件
- 确认外部链接的目标和用途
- 验证依赖的安全性和版本

**类比**：就像安装浏览器扩展，先查看它申请了哪些权限。

#### 第三条：理解执行逻辑

**明确区分**：

- **哪些代码需要运行**：实际执行的脚本
- **哪些只是文档说明**：参考性的文档和注释

**执行前验证**：

- 在沙箱环境测试 Skill 行为
- 监控首次运行的实际操作
- 检查日志和输出是否符合预期

**关键**：理解 Skill 到底会做什么，而不是盲目执行。

### 安全哲学

**核心理念**：安全不是限制能力，而是保护信任。

- **能力与风险并存**：Skill 的强大是双刃剑
- **信任需要验证**：不盲目信任任何 Skill
- **安全支持创新**：在安全边界内发挥 Skill 价值

### 企业级安全实践

#### 技能仓库管理

- **内部仓库**：企业维护的可信 Skill 仓库
- **审核流程**：新 Skill 必须通过安全审查
- **版本控制**：跟踪 Skill 的变更历史

#### 权限隔离

- **最小权限原则**：只授予 Skill 必需的权限
- **沙箱执行**：在隔离环境运行不可信 Skill
- **审计日志**：记录所有 Skill 执行操作

#### 持续监控

- **行为分析**：监控 Skill 的实际行为
- **异常告警**：检测异常的文件访问或网络请求
- **定期审查**：定期审计已安装 Skill 的安全性

### 安全与能力的平衡

**不要**：

- 因为害怕风险就不敢用 Skill
- 盲目信任任何 Skill
- 忽略安全审查

**要**：

- 在安全边界内发挥 Skill 价值
- 验证 Skill 的可信度
- 理解 Skill 的执行逻辑

## 总结

今天我们学习了 Skill 安全三原则：

1. **只安装可信来源的技能**
2. **使用前仔细阅读所有文件**
3. **理解执行逻辑**

**安全哲学**：安全不是限制能力，而是保护信任。

Skill 的强大能力可以帮助我们提升效率、固化流程、放大能力，但前提是在安全边界内使用。

### 课程总结

到这里，我们的 Skill 课程就全部讲完了（共27章 + 导言 + 附录）。

课程分五个部分：

1. **认知篇（01-06章）**：心智模型、渐进式披露、自由度设计
2. **设计篇（07-11章）**：五大设计模式（ToolWrapper、Generator、Reviewer、Inversion、Pipeline）
3. **实战篇（12-19章）**：不写已知知识、Gotchas、文件组织、避免过度约束、内存配置、脚本、Hooks、实战案例
4. **评估篇（20-25章）**：为什么评估、两类分类、七步生命周期、两类评分器、六类评估指标、A/B对比测试
5. **运营与安全篇（26-27章）**：长期维护与团队管理、安全三原则

**核心思想**：

**认知层面**：
- Skill 不是写一次就结束的，而是持续迭代的工作流资产
- 渐进式披露突破上下文限制，按需加载才是正解
- description 写触发条件，不写已知知识，Gotchas 最有价值

**实战层面**：
- 约束目标，不约束路径——给 Agent 足够的自由度
- 脚本封装稳定能力，避免重复造轮子
- Hooks 按需激活、会话隔离，灵活控制临时规则

**评估层面**：
- 没有评估的 Skill 只是假设——从"我相信"变成"我可以证明"
- 六类指标不只是通过率，还要看成本、效率、稳定性
- A/B 对比让变化一目了然，同一个测试集跑两遍

**运营层面**：
- 先让 Skill 进入真实使用，再在反馈里慢慢长出来
- PreToolUse Hook 记录使用情况，数据驱动决策
- 安全保护信任，而不是限制能力

希望这套课程能帮助你在团队中落地 Skill 体系，真正提升研发效能。
