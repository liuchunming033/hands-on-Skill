
> 学习目标：理解Gotchas 坑点的核心概念和实践方法

---


## 引言

你有没有发现，Skill 里写得最详细的部分，往往用不上？

反而是那几条"注意别踩这个坑"，每次都能救命。

Claude Code 核心开发者发现：**任何 Skill 里信号最强的内容，往往不是教程，而是 Gotchas（坑点）。**

## 核心内容

### 为什么 Gotchas 最有价值？

**教程 Agent 本来就知道很多，但坑只有你们团队真的踩过。**

- 教程：网上能搜到，Agent 训练数据里也有
- 坑点：只有你们团队在实战中遇到，Agent 不知道

### 什么是 Gotchas？

Gotchas 是团队在实战中踩过的坑，包括：

| 类型 | 示例 |
|------|------|
| **陷阱** | "这个 API 在 Q4 流量高峰会超时" |
| **边界情况** | "当用户数超过 10 万时，这个查询会变慢" |
| **隐藏依赖** | "这个库需要 Python 3.9+，3.8 会报错" |
| **文档没写的事** | "官方文档说支持批量，但实际超过 100 条会失败" |

### Gotchas 部分该怎么写？

**结构**：

```markdown
## Gotchas（踩坑记录）

### 坑点 1：API 超时问题
- 现象：Q4 流量高峰时 API 会超时
- 原因：服务端限流
- 解决：加重试逻辑，exponential backoff

### 坑点 2：批量操作限制
- 现象：批量超过 100 条会失败
- 原因：服务端硬限制
- 解决：分批次处理，每批最多 100 条
```

### Gotchas 的价值递增

**正向扩充机制**：随着时间推移，Gotchas 会越来越有价值。

- 第一周：2 个坑点
- 第一月：5 个坑点
- 第三月：12 个坑点
- 半年后：20+ 坑点

每踩一个坑，就加一条 Gotchas。Skill 的价值随着时间递增。

### Anthropic 案例：PDF Skill

PDF Skill 在处理非填充式表单时会出问题。

**如果写成教程**：教 Agent 怎么处理 PDF（Agent 本来就会）

**如果写成 Gotcha**："注意：非填充式表单需要用不同的提取方法"

后者才是真正有价值的信息。

### Gotchas vs 教程

| 内容类型 | 价值 | 来源 |
|---------|------|------|
| 教程 | 低 | 网上能搜到 |
| 最佳实践 | 中 | 通用经验 |
| Gotchas | 高 | 团队实战踩坑 |

**真正值钱的是告诉它哪里别踩。**

## 总结

今天我们学习了 Gotchas 坑点：

1. **Gotchas 是信号最强的内容**，比教程更有价值
2. **教程 Agent 本来就知道**，但坑只有你们踩过
3. **正向扩充机制**：每踩一个坑就加一条，价值随时间递增

**核心原则**：真正值钱的是告诉它哪里别踩。
