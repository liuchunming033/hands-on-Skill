
> 学习目标：理解两类 Skill 分类的核心概念和实践方法

---


## 引言

上一节我们理解了为什么需要评估：两个维护面可以独立失控，没有 Eval 只能猜。

但有了评估还不够，**方向错了，评估就白做了**。

你是不是也遇到过这种情况：

- 写了个 Skill，测了半天发现模型自己就能做，这个 Skill 没意义
- 改了个 Skill，一直盯着"是否按你的流程做"，但这个 Skill 的价值在于让模型学会做新事，测错重点了
- 模型升级了，Skill 该不该退役？测错了重点，越测越懵

**根本原因**：Skill 分两大类，两类 Skill 的评估重点完全不同。

如果你分不清这两类，评估方向就错了。

## 核心内容

### 为什么需要分类？

Skill 接入真实工作流后，会跑命令、改文件、留下痕迹。改任何一个地方，行为都可能变。

但不同类型的 Skill，评估要回答的问题不一样。

---

## 📚 两类 Skill 的定义

### 第一类：补能力（Capability Uplift Skills）

**简单说**：教 Agent 原本不会做、容易做错的事。

**典型例子**：

- 处理某类特殊的 PDF（比如非填充式表单）
- 生成某种固定格式的文档
- 跑一套特定的分析流程

**关键问题**：模型现在还做不到、或者做得不够稳，所以需要 Skill 帮它。

**Eval 要回答**：

- 这个 Skill 现在还有用吗？
- 模型升级后，是不是自己就能做了？

---

### 第二类：固化偏好（Encoded Preference Skills）

**简单说**：固化你的工作流程、判断标准、个人偏好。

**典型例子**：

- 代码审查永远先看安全边界
- 写研究报告时必须先区分事实推断和假设
- 每次生成视频稿都按你自己的叙事节奏

**关键问题**：模型自己能做这件事，但你希望它按你的方式做。

**Eval 要回答**：

- 它还在按你的流程工作吗？
- 它还在用你的标准判断吗？

---

## 💡 两类 Skill 的关键差异

### 补能力类的问题：模型会进步

今天必须靠 Skill 才能做的事，下一个模型版本可能自己就能做了。

**例子**：处理某类 PDF 的 Skill。

- Agent 3.5 版本：必须靠 Skill 才能处理非填充式表单
- Agent 4.0 版本：自己可能就能处理了

这时候，Skill 的价值就变弱了，甚至该退役了。

**评估重点**：证明 Skill 有增量。

- 对比有 Skill 和无 Skill 的效果
- 如果有 Skill 明显更稳、更快、更省 token，就证明了自己有用
- 如果不用 Skill 也一样好，Skill 可能该退役

---

### 固化偏好类的特点：价值在于忠实度

即使模型自己能做，但你希望它按你的流程、你的标准、你的偏好来做。

**例子**：代码审查 Skill。

- Agent 自己能审查代码，但你希望它永远先看安全边界
- Agent 自己能写研究报告，但你希望它先区分事实推断和假设

**评估重点**：检查忠实度。

- 它还在执行你的流程吗？
- 它还在用你的判断标准吗？
- 它还在遵循你的偏好吗？

---

## ✅ 两类 Skill 的共同点

**没有 Eval，你都只能猜。**

凭感觉改 Skill 时，两类 Skill 都会遇到同样的困惑：

- 怎么知道真的变好了？
- 怎么知道不是只在这个例子上好用？
- 怎么知道没有把别处搞坏？

---

## 🔧 如何判断 Skill 属于哪一类？

问自己一个问题：

**"这个 Skill 是让模型能做原本不会做的事，还是让模型按我的方式做事？"**

- 前者 → 补能力类
- 后者 → 固化偏好类

---

## 📊 评估重点差异

| Skill 类型 | 评估重点 | 评估方法 |
|-----------|---------|---------|
| **补能力类** | 证明有增量 | 对比有 Skill vs 无 Skill，看模型升级后是否还能用 |
| **固化偏好类** | 检查忠实度 | 看是否执行你的流程、是否用你的标准 |

---

## 💡 为什么这个分类很重要？

**搞错了类型，评估就白做了。**

如果你把固化偏好类当成补能力类来评估：

- 你会关注"模型能不能做"，但这不是重点
- 你忽略了"是否按你的方式做"，这才是核心

如果你把补能力类当成固化偏好类来评估：

- 你会关注"忠实度"，但这不是重点
- 你忽略了"增量"，这才是 Skill 该不该存在的依据

**方向错了，评估就没意义。**

## 🎯 本节核心观点

今天我们学习了 Skill 的两类分类：

- **补能力类**（Capability Uplift）：教模型做原本不会的事，重点是证明有增量
- **固化偏好类**（Encoded Preference）：让模型按你的方式做事，重点是检查忠实度

两类 Skill 的共同点：**没有 Eval，你都只能猜**。

**核心原则**：分清类型，找准方向，评估才有意义。

方向对了，评估才有意义。
